In May 2026, we released [SmithDB](https://www.langchain.com/blog/introducing-smithdb?utm_source=docs), a new observability database built from the ground up for modern AI agents. Agent traces have grown dramatically in complexity: hundreds of nested spans, multi-modal content, and spans that remain open for hours. General-purpose databases were never designed to handle these patterns at scale. SmithDB addresses this directly, built in Rust on Apache DataFusion and Vortex with purpose-built support for the query patterns that matter most for agent observability: random access, interactive filtering, full-text search, and tree-aware trace reconstruction.

The response from customers has been outstanding. SmithDB delivers leading performance across the key workloads that matter for agent observability and makes core LangSmith experiences up to 12x faster than before.

Today we are excited to bring SmithDB-powered methods to all SDK users.

<Info>

Minimum SDK version requirements will be published at general availability. We are shipping new methods every week, building toward full feature parity with the existing methods before a general availability release and broader customer communication.

These methods ship in minor SDK releases today. When a future v2 major release completes the transition, code already using these methods will require no further changes.

Your feedback matters. Reach out in Slack or open a support ticket with questions or issues so we can address them quickly and make this migration as seamless as possible.

## Runs

### Query runs

Query runs from a project with optional filtering and field projection. Returns a paginated result set—see [Pagination changes](#common-patterns) for important behavior differences from the existing query methods.

#### Code migration

<Tabs>
  <Tab title="Python">
    <Note>Requires `langsmith>=0.9.2`.</Note>

    <CodeGroup>
    ```python Before
    from langsmith import Client

    client = Client()

    runs = client.list_runs(
        project_name="my-project",
        run_type="llm",
        filter='and(eq(status, "success"), gt(total_tokens, 100))',
        limit=100,
    )

    for run in runs:
        print(run.id, run.name, run.status)
    ```

    ```python After
    import asyncio

    from langsmith import Client


    async def main():
        client = Client()

        async for run in client.runs.query(
            project_name="my-project",
            run_type="LLM",
            filter='and(eq(status, "success"), gt(total_tokens, 100))',
            selects=["ID", "NAME", "STATUS"],
        ):
            print(run["ID"], run["NAME"], run["STATUS"])


    asyncio.run(main())
    ```
    </CodeGroup>

**List all runs in a project**

**Selecting fields**

`list_runs` returns a default set of fields without requiring explicit selection. `runs.query` returns only `id` by default — you must declare every field you need via `selects=[...]`. Two renames: `select` → `selects` (plural), and field names are now **uppercase** (`"name"` → `"NAME"`).

<Tabs>
  <Tab title="Python">

<CodeGroup>

```python Before
from langsmith import Client

client = Client()

# returns a default set of fields; no explicit selection needed
runs = client.list_runs(project_name="my-project")
```

```python After
import asyncio

from langsmith import Client


async def main():
    client = Client()

    project = await asyncio.to_thread(client.read_project, project_name="my-project")
    # must explicitly list every field needed; default returns only id
    async for run in client.runs.query(
        project_ids=[str(project.id)],
        selects=["ID", "NAME", "RUN_TYPE", "STATUS", "START_TIME", "INPUTS", "ERROR"],
    ):
        ...


asyncio.run(main())
```

**Pagination**

`list_runs` auto-paginates transparently — the SDK fetches 100 runs per API call and advances the cursor internally. Use `itertools.islice()` to stop iteration client-side. `runs.query` exposes pagination explicitly: iterate with `async for` and a `break` to stop. For page-level control, `runs.query` also provides `page.has_next_page()` and `await page.get_next_page()`.

<Tabs>
  <Tab title="Python">

<CodeGroup>

```python Before
import itertools

from langsmith import Client

client = Client()

runs = itertools.islice(
    client.list_runs(project_name="my-project"),
    150,
)
```

```python After
import asyncio

from langsmith import Client


async def main():
    client = Client()

    project = await asyncio.to_thread(client.read_project, project_name="my-project")
    runs = []
    async for run in client.runs.query(
        project_ids=[str(project.id)],
        selects=["ID", "NAME"],
    ):
        runs.append(run)
        if len(runs) >= 150:
            break


asyncio.run(main())
```

**Scoped filters: filter, trace_filter, tree_filter**

All three filter parameters are unchanged between v1 and v2. `filter` applies to the matched run, `trace_filter` to the root of its trace, and `tree_filter` to other runs in the trace tree (siblings and children).

#### Parameter renames

<Warning>
`project_name` is not supported in `runs.query`. Pass `project_ids` with the project UUID instead. To look up a UUID by name: `client.read_project(project_name="my-project").id`
</Warning>

<Warning>
`min_start_time` defaults to **1 day ago** when omitted. `list_runs` with no `start_time` returned all historical runs; `runs.query` without `min_start_time` silently scopes the query to the last 24 hours. Pass an explicit `min_start_time` if you need a wider window.
</Warning>

#### Common patterns

**Fetch a single run by ID**

v2 requires two additional parameters that `read_run` did not need: `project_id` (UUID) and `start_time`. Both are required by the SmithDB storage model to locate a run efficiently. Resolve the project UUID via `read_project()` first.

<Tabs>
  <Tab title="Python">

<CodeGroup>

```python Before
from langsmith import Client

client = Client()

run = client.read_run("<run-id>")
```

```python After
import asyncio

from langsmith import Client


async def main():
    client = Client()

    project = await asyncio.to_thread(client.read_project, project_name="my-project")
    run = await client.runs.retrieve(
        run_id="<run-id>",
        project_id=str(project.id),
        start_time="2025-01-01T12:00:00Z",
    )


asyncio.run(main())
```

#### Parameter renames

<Warning>
`runs.retrieve` requires two new fields—`project_id` and `start_time`—that `read_run` did not need. Both are required to locate the run in SmithDB.
</Warning>

<Tabs>
  <Tab title="Python">
    All parameters use `snake_case`.

    | Before (`read_run`) | After (`runs.retrieve`) | Notes |
    |---|---|---|
    | `run_id` | `run_id` | Unchanged |
    | `load_child_runs` | *(removed)* | No equivalent |
    | *(not available)* | `project_id` | **Required**—UUID of the project that owns the run |
    | *(not available)* | `start_time` | **Required**—run's start time (RFC3339), used with `project_id` to locate the run |
    | *(all fields returned by default)* | `selects` | Field projection; defaults to `["ID"]` only; field names are uppercase |
  </Tab>
  <Tab title="TypeScript">
    The v2 API uses `snake_case` parameter names, unlike v1 which used `camelCase`.

    | Before (`readRun`) | After (`client.runs.retrieveV2`) | Notes |
    |---|---|---|
    | `runId` | `runId` | Unchanged (positional parameter) |
    | `options.loadChildRuns` | *(removed)* | No equivalent |
    | *(not available)* | `project_id` | **Required**—`snake_case`; UUID of the project that owns the run |
    | *(not available)* | `start_time` | **Required**—`snake_case`; run's start time (RFC3339), used to locate the run |
    | *(all fields returned by default)* | `selects` | Field projection; defaults to `["ID"]` only; field names are uppercase |
  </Tab>
  <Tab title="Java">
    All parameters use `camelCase` builder methods.

    | Before (`RunRetrieveParams`) | After (`RunRetrieveV2Params`) | Notes |
    |---|---|---|
    | `runId()` | `runId()` | Unchanged |
    | `sessionId()` | *(removed)* | Replaced by `projectId()` |
    | `startTime()` | `startTime()` | Now **required**; used with `projectId()` to locate the run in SmithDB |
    | `excludeS3StoredAttributes()` | *(removed)* | No equivalent |
    | `excludeSerialized()` | *(removed)* | No equivalent |
    | `includeMessages()` | *(removed)* | No equivalent |
    | *(not available)* | `projectId()` | **Required**—UUID of the project that owns the run |
    | *(all fields returned by default)* | `selects()` | Field projection; defaults to `["ID"]` only; field names are uppercase |
  </Tab>

    Go SmithDB v2 support is planned. Check back for updates.
  </Tab>
  <Tab title="cURL">
    Query parameter names match the Python `snake_case` names exactly.
